3  Oggetti e classi

3.1 Cos’è un oggetto?

Nella vita reale siamo circondati da oggetti: una sedia, un telefono, una persona. Ma anche in un videogioco esistono oggetti: un personaggio, un nemico, un’arma, un livello. Ognuno di questi ha due caratteristiche fondamentali:

  • delle proprietà che lo descrivono (nome, vita, danno, livello…)
  • dei comportamenti che sa mettere in atto (attaccare, difendersi, raccogliere oggetti…)

Nell’OOP usiamo esattamente questa stessa idea. Un oggetto è un’entità che combina:

  • attributi — i dati che descrivono lo stato dell’oggetto
  • metodi — le azioni che l’oggetto sa compiere
NoteDefinizione

Un oggetto è un’istanza concreta che raggruppa dati (attributi) e comportamenti (metodi) in un’unica entità coerente.

3.2 Cos’è una classe?

Gli oggetti non nascono dal nulla: vengono creati a partire da uno schema, chiamato classe.

Una classe è come la scheda di un personaggio in un GDR (gioco di ruolo): descrive quali statistiche ha ogni personaggio di quel tipo e cosa sa fare. Ma la scheda non è il personaggio — dal modello si possono creare tanti personaggi diversi, ognuno con i propri valori.

TipClasse vs oggetto — l’analogia del GDR

La classe Guerriero è la scheda tipo: dice che ogni guerriero ha vita, forza, armatura e sa attaccare() e difendersi().

Gli oggetti guerriero1, guerriero2, boss sono i personaggi reali: creati dallo stesso schema, ma con valori diversi.

In sintesi:

Descrizione Esempio
Classe Il modello, lo schema generale Guerriero
Oggetto La realizzazione concreta guerriero1

Una classe può generare quanti oggetti si vuole, ognuno con i propri valori.

3.3 Esempio 1: il personaggio

Vogliamo modellare un personaggio di un videogioco.

Cosa descrive un personaggio?

  • nome — il nome del personaggio
  • vita — i punti vita attuali
  • attacco — il danno che infligge
  • livello — il livello attuale

Cosa sa fare un personaggio?

  • subirDanno(int danno) — riduce la vita in modo controllato
  • attacca(Personaggio bersaglio) — infligge danno a un altro personaggio
  • salaDiLivello() — aumenta il livello e migliora le statistiche

In Java, la classe si scrive così:

public class Personaggio {

    // Attributi
    String nome;
    int vita;
    int attacco;
    int livello;

    // Subisce danno (la vita non scende sotto zero)
    void subirDanno(int danno) {
        vita -= danno;
        if (vita < 0) vita = 0;
        System.out.println(nome + " subisce " + danno + " danni. Vita rimasta: " + vita);
    }

    // Attacca un altro personaggio
    void attacca(Personaggio bersaglio) {
        System.out.println(nome + " attacca " + bersaglio.nome + "!");
        bersaglio.subirDanno(attacco);
    }

    // Sale di livello
    void salaDiLivello() {
        livello++;
        attacco += 5;
        vita += 20;
        System.out.println(nome + " sale al livello " + livello + "!");
    }
}

Ora possiamo creare oggetti a partire da questa classe:

Personaggio eroe = new Personaggio();
eroe.nome = "Kratos";
eroe.vita = 100;
eroe.attacco = 25;
eroe.livello = 1;

Personaggio nemico = new Personaggio();
nemico.nome = "Drago";
nemico.vita = 200;
nemico.attacco = 40;
nemico.livello = 5;

// Usiamo i metodi
eroe.attacca(nemico);    // "Kratos attacca Drago! Drago subisce 25 danni. Vita rimasta: 175"
nemico.attacca(eroe);    // "Drago attacca Kratos! Kratos subisce 40 danni. Vita rimasta: 60"
eroe.salaDiLivello();    // "Kratos sale al livello 2!"

I due personaggi derivano dalla stessa classe, ma sono oggetti indipendenti: modificare la vita di uno non influenza l’altro.

3.4 Esempio 2: l’arma

Un esempio più specifico: vogliamo modellare un’arma che si consuma con l’uso, come in molti RPG.

Attributi:

  • nome — il nome dell’arma
  • dannoBase — il danno che infligge
  • durabilita — i colpi rimasti prima che si rompa

Metodi:

  • usa() — usa l’arma, riducendo la durabilità
  • ripara() — ripristina la durabilità
  • getDanno() — restituisce il danno attuale (che cala con la durabilità)
public class Arma {

    // Attributi
    String nome;
    int dannoBase;
    int durabilita;

    // Usa l'arma: scala la durabilità
    void usa() {
        if (durabilita > 0) {
            durabilita--;
            System.out.println(nome + " usata. Durabilità rimasta: " + durabilita);
        } else {
            System.out.println(nome + " è rotta! Devi ripararla.");
        }
    }

    // Ripara l'arma
    void ripara() {
        durabilita = 100;
        System.out.println(nome + " riparata!");
    }

    // Il danno cala proporzionalmente alla durabilità
    int getDanno() {
        return (dannoBase * durabilita) / 100;
    }
}

Usiamo l’arma con un personaggio:

Arma spada = new Arma();
spada.nome = "Spada del Caos";
spada.dannoBase = 50;
spada.durabilita = 100;

System.out.println("Danno: " + spada.getDanno()); // Danno: 50
spada.usa();   // Durabilità rimasta: 99
spada.usa();   // Durabilità rimasta: 98
System.out.println("Danno: " + spada.getDanno()); // Danno: 49
ImportantOsserva il metodo usa()

Prima di ridurre la durabilità, il metodo controlla che l’arma non sia già rotta. Con una variabile globale durabilita accessibile da tutti, nessuno impedirebbe di portarla sotto zero. L’oggetto invece gestisce i propri dati in modo responsabile.

3.5 La parola chiave new

Ogni volta che creiamo un oggetto usiamo la parola chiave new:

Personaggio eroe = new Personaggio();
Arma spada = new Arma();

new fa tre cose:

  1. alloca la memoria necessaria per l’oggetto
  2. inizializza gli attributi con valori di default
  3. restituisce un riferimento all’oggetto appena creato

La variabile eroe non contiene l’oggetto direttamente, ma un riferimento — un indirizzo che punta a dove l’oggetto è memorizzato. Questo concetto diventerà importante più avanti.

3.6 Accedere ad attributi e metodi

Per accedere agli attributi e ai metodi di un oggetto si usa il punto .:

// Accesso a un attributo
eroe.vita = 100;
int danno = spada.getDanno();

// Chiamata a un metodo
eroe.salaDiLivello();
spada.usa();
eroe.attacca(nemico);

La notazione con il punto si legge come: “sull’oggetto X, esegui l’azione Y” oppure “dell’oggetto X, leggi la proprietà Y”.

3.7 Riepilogo

NoteI concetti chiave di questa lezione
  • Una classe è un modello che descrive attributi e metodi di un tipo di oggetto.
  • Un oggetto è una realizzazione concreta di una classe, con i propri valori.
  • Da una classe si possono creare quanti oggetti si vuole, ognuno indipendente dagli altri.
  • Si accede ad attributi e metodi con la notazione punto: oggetto.attributo, oggetto.metodo().
  • La parola chiave new crea un nuovo oggetto in memoria.

Nella prossima lezione vedremo come proteggere gli attributi di un oggetto e perché è fondamentale farlo — è il cuore dell’incapsulamento.
